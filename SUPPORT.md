# Support

## Where to Start

- Read the main overview in [README.md](README.md)
- Browse the docs index in [docs/README.md](docs/README.md)
- Review deployment and release guidance in `docs/release/`

## Questions and Usage Help

Open a GitHub issue when:

- a command is unclear
- behavior is confusing
- documentation is missing
- you need guidance on configuration or deployment

Please include:

- what you are trying to do
- the command or workflow you used
- the relevant environment details
- the output or failure you saw

## Bugs

Use the bug report template for:

- regressions
- broken commands
- dashboard failures
- runtime or service issues

## Security and Sensitive Reports

Do not post vulnerabilities, secrets, or private user data publicly.

Use the process described in [SECURITY.md](SECURITY.md).

## Project Scope

Cadiax currently maintains public branding as `Cadiax`, while keeping internal compatibility surfaces such as `otonomassist`, `OTONOMASSIST_*`, and `.otonomassist/`.
